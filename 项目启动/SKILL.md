# /start - 一站式项目创建器

$ARGUMENTS

---

## 🚀 概述

此指令启动 **终极项目协议 (Ultimate Project Protocol)**。它将策略、设计和执行统一到一个单一的自动化流水线中。

**工作流阶段：**

1. **概念与策略** (头脑风暴 + 选择)
2. **深度规划** (任务拆解 + 角色分配)
3. **专业优化** (AI/文档/数据库/SEO 注入)
4. **设计注入** (UI/UX Pro Max)
5. **多智能体执行** (编排 + 整洁代码)
6. **交付** (部署准备 + 调试)

---

## 1. 概念与策略 (头脑风暴)

**动作：**
运行 `/brainstorm "$ARGUMENTS"`

**目标：**

- 理解核心问题。
- 生成 3 种不同的架构/产品方法。
- **用户决定：** 选择最佳方法。

> 🛑 **停止：** 在继续之前等待用户选择。

---

## 2. 深度规划 (生成计划)

**动作：**
运行 `/plan "Selected Option Details"`

**目标：**

- 创建 `docs/PLAN-{slug}.md`。
- 分析需求以识别必要的智能体。
- **角色分配：** 从 `orchestrate.md` 矩阵中选择至少 3 个智能体 (例如：前端、后端、安全)。

---

## 3. 系统架构蓝图 (架构师)

**动作：**
在写代码*之前*建立技术基础。

**步骤：**

1. **API 设计：**
    - 加载技能：`API设计模式`
    - 定义：REST vs GraphQL, 端点结构, 响应格式。
2. **数据库建模：**
    - 加载技能：`数据库建模`
    - 设计：ER 图, Schema 定义。
3. **技术决策：**
    - 加载技能：`系统架构设计`
    - 输出：`docs/ARCHITECTURE.md` (技术栈, 权衡, ADRs)。

---

## 4. 专业优化 (技能注入)

**动作：**
根据项目类型注入专业技能。

### 🤖 AI 集成开发 (条件性)
>
> **触发：** 如果请求包含 "AI", "LLM", "Gemma", "Search", "Analysis"

- **加载技能：** `gemma3调用流程` (Gemma 3 集成)
- **加载技能：** `ddgs网络搜索` (实时搜索)
- **目标：** 确保从一开始就内置 AI 能力。

### 🗄️ 数据库设计 (条件性)
>
> **触发：** 如果项目需要数据持久化

- **加载技能：** `数据库建模` (Database Modeling)
- **目标：** 编码前设计 Schema。

### 📈 SEO 策略 (条件性)
>
> **触发：** 如果项目是 Web 应用 / 网站

- **加载技能：** `SEO基础` (SEO Fundamentals)
- **目标：** 生成 SEO 检查清单和 Meta 策略。

### 📄 文档 (强制性)

- **加载技能：** `文档模板` (Documentation Templates)
- **目标：** 初始化 `README.md` 和标准 `docs/` 结构。

---

## 5. 设计注入 (UI/UX)

**动作：**
运行 `/ui-ux-pro-max`

**目标：**

- 搜索与定义：调色板、排版、设计系统。
- 生成：`docs/DESIGN_SPECS.md` (或追加到 PLAN)。
- **输出：** 为前端智能体提供清晰的视觉指南。

---

---

## 6. 多智能体执行 (编排)

**动作：**
运行 `/orchestrate` (阶段 2 - 实施)

**强制规则：**

1. **并行执行：** 启动已识别的智能体 (前端、后端等)。
2. **整洁代码协议：** 对**所有**生成的代码强制执行 `整洁代码规范` (ESLint/Prettier)。
3. **上下文传递：**
   - 前端智能体 <== 接收 `docs/DESIGN_SPECS.md`
   - 后端智能体 <== 接收 `docs/PLAN-{slug}.md` + Schema

**验证：**

- 运行：`python ~/.agent/skills/lint-and-validate/scripts/lint_runner.py .`

---

---

## 7. 交付与润色

**动作：**
完成项目以进行移交。

1. **部署准备：**
   - 加载技能：`部署流程` (Deployment Procedures)。
   - 检查是否需要 `Dockerfile`, `vercel.json` 等。

2. **演示：**
   - 创建 `docs/WALKTHROUGH.md` 演示功能。

3. **调试 (如果需要)：**
   - 如果仍有错误，建议 `/debug`。

---

## 使用示例

```bash
/start Create a crypto dashboard with AI prediction
```

**发生什么：**

1. 头脑风暴 3 个仪表盘概念。
2. 用户选择 "Real-time D3.js Dashboard"。
3. 规划任务。
4. **架构系统：** 定义 API 统计端点 & DB Schema。
5. 加载 `gemma3` + `ddgs` (因为涉及 "AI")。
6. 加载 `web-design` 技能。
7. 根据特定设计规范构建智能体。
8. 交付可工作的代码。
